# Действия бота после исполнения всех SELL ордеров

## 🔄 Автоматическая ребалансировка SELL сетки

### Триггер
Когда исполняется последний SELL ордер, автоматически вызывается функция `check_rebalancing()`.

---

## 📋 Последовательность действий

### ШАГ 1: Проверка условий

**Условия для запуска ребалансировки:**
```
✓ len(open_sell_orders) == 0  (все SELL ордера закрыты)
✓ state == TRADING            (бот в режиме торговли)
```

**Где проверяется:**
- В функции `handle_sell_filled()` после обработки каждого SELL ордера
- Вызывается `await self.check_rebalancing(current_price)`

---

### ШАГ 2: Расчет суммы для рыночной покупки

**Формула:**
```
market_buy_amount = (5 × buy_order_value) + 2 USDT
```

**Примеры:**
- Если `buy_order_value = 10 USDT`:
  ```
  market_buy_amount = (5 × 10) + 2 = 52 USDT
  ```
- Если `buy_order_value = 20 USDT`:
  ```
  market_buy_amount = (5 × 20) + 2 = 102 USDT
  ```
- Если `buy_order_value = 50 USDT`:
  ```
  market_buy_amount = (5 × 50) + 2 = 252 USDT
  ```

**Почему +2 USDT?**
- Резерв для точного размещения ордеров
- Компенсация комиссий и округлений
- Гарантия достаточного баланса для создания 5 ордеров

---

### ШАГ 3: Проверка баланса

**Проверка:**
```
if quote_balance >= market_buy_amount:
    → Выполняется рыночная покупка
else:
    → Логируется предупреждение
    → Ребалансировка не выполняется
```

**Если баланса недостаточно:**
- Логируется предупреждение: `"Insufficient balance for market buy"`
- Бот продолжает работу с существующими BUY ордерами
- SELL ордера не создаются до следующего исполнения BUY

---

### ШАГ 4: Рыночная покупка

**Выполнение:**
1. Используется метод `place_market()` с параметром `quoteOrderQty`
   - Покупка на сумму, а не на количество
   - Точное соответствие требуемой сумме

2. Ожидание исполнения:
   ```
   await asyncio.sleep(3)  # 3 секунды
   ```

3. Получение информации об ордере:
   - `executed_price` - цена исполнения
   - `executed_qty` - количество купленного актива

4. Добавление позиции в FIFO:
   ```
   position_manager.add_position(executed_price, executed_qty)
   ```
   - Позиция добавляется с временной меткой
   - Используется для расчета прибыли при следующей продаже

---

### ШАГ 5: Обновление балансов

**Обновление:**
```
base_asset = balance(base_asset_name)      # Обновленный баланс базового актива
current_deposit = balance(quote_asset_name) # Обновленный баланс USDT
```

**Зачем:**
- Точные данные для создания SELL ордеров
- Корректный расчет доступного количества

---

### ШАГ 6: Создание новых SELL ордеров

**Функция:** `create_sell_grid_only(current_price)`

**Алгоритм:**

1. **Расчет доступного баланса:**
   ```
   available_base_asset = current_base_balance - locked_base_asset
   ```
   - `locked_base_asset` = сумма всех открытых SELL ордеров (обычно 0)

2. **Расчет количества на ордер:**
   ```
   sell_qty_per_order = available_base_asset / 5
   ```
   - Весь доступный баланс делится на 5 равных частей

3. **Расчет цены первого ордера:**
   ```
   first_sell_price = current_price × (1 + grid_step_pct)
   ```
   - Первый ордер размещается на шаг сетки выше текущей цены
   - Пример: если цена 100 USDT, шаг 0.75% → первый ордер на 100.75 USDT

4. **Создание 5 SELL ордеров:**
   ```
   Order 1: first_sell_price
   Order 2: first_sell_price × (1 + grid_step_pct)
   Order 3: order_2_price × (1 + grid_step_pct)
   Order 4: order_3_price × (1 + grid_step_pct)
   Order 5: order_4_price × (1 + grid_step_pct)
   ```

5. **Проверки перед размещением:**
   - `qty >= minQty` (минимальное количество)
   - `notional >= minNotional` (минимальный номинал)
   - Цена округляется до `tickSize`
   - Количество округляется до `stepSize`

6. **Размещение ордеров:**
   - Каждый ордер размещается с задержкой 0.2 секунды
   - После размещения добавляется в список `self.orders`

---

### ШАГ 7: Сохранение состояния

**Действие:**
```
self.save_state()
```

**Что сохраняется:**
- Все новые SELL ордера
- Обновленные балансы
- Позиции в FIFO
- Текущее состояние бота

---

## 📊 Пример полного цикла

### Сценарий:
- Текущая цена: 100 USDT
- Шаг сетки: 0.75%
- Размер ордера: 10 USDT
- Баланс USDT: 100 USDT

### Действия:

1. **Все 5 SELL ордеров исполнены**
   - Прибыль рассчитана и добавлена в `profit_bank`
   - Вызывается `check_rebalancing()`

2. **Расчет суммы покупки:**
   ```
   market_buy_amount = (5 × 10) + 2 = 52 USDT
   ```

3. **Проверка баланса:**
   ```
   100 USDT >= 52 USDT ✓ (достаточно)
   ```

4. **Рыночная покупка:**
   - Покупается на 52 USDT
   - Получено, например: 0.52 токена по цене 100 USDT
   - Позиция добавлена в FIFO

5. **Создание новых SELL ордеров:**
   ```
   Order 1: 100.75 USDT (qty = 0.104)
   Order 2: 101.51 USDT (qty = 0.104)
   Order 3: 102.27 USDT (qty = 0.104)
   Order 4: 103.04 USDT (qty = 0.104)
   Order 5: 103.81 USDT (qty = 0.104)
   ```

6. **Результат:**
   - Создана новая SELL сетка
   - Бот готов к следующему циклу
   - BUY ордера продолжают работать

## ⚠️ Особые случаи

### Случай 1: Недостаточно баланса

**Ситуация:**
```
quote_balance = 30 USDT
market_buy_amount = 52 USDT
30 < 52 → недостаточно
```

**Действия:**
- Рыночная покупка НЕ выполняется
- SELL ордера НЕ создаются
- Логируется предупреждение
- Бот продолжает работу с BUY ордерами
- При следующем исполнении BUY создастся SELL ордер

### Случай 2: Ошибка рыночной покупки

**Возможные ошибки:**
- "Permission denied" → критическая ошибка, бот останавливается
- "balance not enough" → логируется предупреждение, продолжается работа
- Другие ошибки → логируется, продолжается работа

**Действия:**
- SELL ордера не создаются
- Бот продолжает работу
- При следующем исполнении SELL попытка повторится

### Случай 3: Недостаточно базового актива после покупки

**Ситуация:**
- Рыночная покупка выполнена
- Но полученного количества недостаточно для создания 5 ордеров

**Действия:**
- Создаются столько SELL ордеров, сколько возможно
- Остаток баланса остается неиспользованным
- При следующем исполнении BUY будет использован

## 🔄 Интеграция с другими процессами

### Связь с Pyramiding

**После ребалансировки:**
- Если есть `profit_bank >= buy_order_value`
- Автоматически проверяется возможность создания нового BUY ордера
- Новый BUY размещается ниже самой низкой цены в сетке

### Связь с основным циклом

**В основном цикле (`main_loop`):**
- Каждые 15 секунд проверяются ордера
- Если SELL ордера исполнены → автоматически вызывается ребалансировка
- Процесс полностью автоматический

### Связь с критической ситуацией

**Если после ребалансировки:**
- Все BUY ордера исполнены
- SELL ордера не созданы (недостаточно баланса)
- → Срабатывает `check_critical_situation()`
- → Отправляется критическое уведомление

## 📈 Преимущества автоматической ребалансировки

1. **Непрерывность работы:**
   - Бот всегда имеет SELL ордера для фиксации прибыли
   - Не требуется ручное вмешательство

2. **Автоматическое реинвестирование:**
   - Прибыль от продаж автоматически используется для новых покупок
   - Создается новый цикл торговли

3. **Оптимальное использование баланса:**
   - Резерв 2 USDT гарантирует точное размещение
   - Весь доступный баланс используется эффективно

4. **Сохранение стратегии:**
   - SELL ордера всегда размещаются на шаг сетки выше
   - Сохраняется логика Grid Trading

```
Все SELL ордера исполнены
        ↓

        ↓
    [ДА] → Рыночная покупка на (5×ордер + 2 USDT)
        ↓
    Получение базового актива
        ↓
    Добавление в FIFO
        ↓
    Создание 5 новых SELL ордеров
        ↓
    Сохранение состояния
        ↓
    Продолжение работы
    
    [НЕТ] → Логирование предупреждения
        ↓
    Продолжение работы с BUY ордерами
        ↓
    При следующем BUY создастся SELL
```
