# Поиск свободного уровня сетки (якорь + гибрид)

Документ описывает, как бот выбирает **цену** для нового лимитного ордера, когда «идеальный» уровень по шагу сетки уже занят другим открытым ордером. Реализация: `TradingBot.find_next_free_buy_price_down` / `find_next_free_sell_price_up` в **`trading_bot.py`**, константы — **`config.py`**, тесты — **`tests/test_grid_free_levels.py`**.

---

## Зачем это нужно

Раньше при занятой цене (дубликат уровня) новый ордер мог **не выставляться** или использовался узкий запасной сценарий. Сейчас бот **не пропускает** размещение без необходимости: сначала ищет свободный уровень **строго по шагу сетки** в нужную сторону, затем при необходимости — **мелкие отступы** от **якорной** цены (доля меньше основного `grid_step_pct`), чтобы не уводить цену слишком далеко от логики сетки.

Учитываются только ордера со статусом **`open`**. Цены приводятся к **`tickSize`** (`_align_to_tick`).

---

## Якорь (anchor)

Функции принимают **якорную цену события**, а не «уже сдвинутый на один шаг» уровень.

| Направление | Смысл якоря | Первый кандидат по сетке |
|-------------|----------------|---------------------------|
| **BUY вниз** (`find_next_free_buy_price_down`) | Цена, **от которой** считается сетка вниз (исполненный SELL, самый нижний открытый BUY в пирамидинге, предыдущий BUY при `create_buy_after_buy`) | `anchor × (1 − grid_step_pct)` |
| **SELL вверх** (`find_next_free_sell_price_up`) | Цена **после BUY** (fill покупки) или исполненного SELL при цепочке SELL | `anchor × (1 + grid_step_pct)` |

Передавать в якорь уже вычисленный «первый уровень» **нельзя** — получится двойной шаг.

---

## Алгоритм (гибрид)

1. **Сетка:** от якоря считается первый уровень по формуле выше; если на этом **тике** уже есть открытый ордер нужной стороны — переход к следующему уровню: умножение текущей цены на `(1 ∓ grid_step_pct)` и снова выравнивание по тику. Так повторяется не более **`GRID_FREE_MAX_STEPS`** раз (переменная окружения, по умолчанию **5**, см. `config.py`).

2. **Мелкие отступы:** если свободный уровень по сетке не найден, перебираются доли **`fb`**, строго **меньшие** основного шага сетки, из списков:
   - шаг **1.5%** → `GRID_FALLBACK_BUY_BELOW_ANCHOR_PCT_015`;
   - шаг **0.75%** → `GRID_FALLBACK_BUY_BELOW_ANCHOR_PCT_0075`;
   - иначе → из `GRID_FALLBACK_BELOW_ANCHOR_PCT_GENERIC` отбираются только `fb < grid_step_pct`.

   Для BUY: `anchor × (1 − fb)`; для SELL: `anchor × (1 + fb)`.

3. Если и после этого свободного уровня нет — возвращается **`None`**, вызывающий код логирует отказ и обычно **не** создаёт ордер.

---

## Где вызывается

- **`create_buy_after_sell(sell_price)`** — якорь = цена исполненного SELL.
- **`create_buy_after_buy`** — якорь = цена исполненного BUY.
- **`check_pyramiding`** — якорь = цена **самого нижнего** открытого BUY (`lowest_order.price`); новый BUY пирамидинга — на свободном уровне **ниже** этой «дна лесенки» (в норме на один шаг сетки ниже, если уровень свободен).
- **`create_sell_after_sell`** — якорь = цена исполненного SELL.
- **`handlers.handle_buy_filled`** (создание SELL после BUY) — якорь = **цена fill BUY**, не заранее посчитанный SELL над ней.

---

## Настройка (.env)

| Переменная | Назначение |
|------------|------------|
| **`GRID_FREE_MAX_STEPS`** | Сколько раз идти по сетке (вниз для BUY / вверх для SELL) перед перебором мелких `fb`. По умолчанию `5`. |

Списки мелких долей заданы в **`config.py`** (при необходимости правятся в коде; не все вынесены в `.env`).
