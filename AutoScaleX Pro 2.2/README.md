# AutoScaleX Pro 2.2

Спотовый торговый бот для BingX с Grid Trading, Pyramiding и DCA стратегиями.

## Особенности

- ✅ **Асинхронная архитектура** — быстрая и эффективная работа
- ✅ **Реферальная система** — доступ только для рефералов админского аккаунта
- ✅ **Полное Telegram управление** — все команды через удобный интерфейс
- ✅ **Автоматическое сохранение состояния** — восстановление после падения
- ✅ **Автоматический перезапуск** — устойчивость к сбоям
- ✅ **FIFO расчёт прибыли** — точный учёт позиций
- ✅ **Grid Trading** — автоматическая сетка ордеров с мультипликативным шагом (~1.5% или 0.75% между уровнями), до 60/125 BUY + 5 SELL
- ✅ **Pyramiding** — расширение сетки за счёт прибыли (новый BUY ниже после каждого SELL при достаточном profit_bank)
- ✅ **DCA** — усреднение позиций при падении цены
- ✅ **Защита большой сетки** — при 3 открытых BUY и большой сетке добавление до 5 BUY внизу (порог: 62 при 1.5%, 127 при 0.75%)
- ✅ **Подготовка к ребалансу** — при 1 оставшемся SELL отмена 5 самых низких BUY для освобождения USDT под новую SELL-сетку
- ✅ **Fallback при занятой цене** — если на основной цене после SELL уже стоит BUY, выставляется BUY по запасному шагу (1% при шаге 1.5%, 0.5% при 0.75%)

## Установка

1. Клонируйте репозиторий или скопируйте файлы
2. Установите зависимости:

**Вариант 1 (рекомендуется для Windows):**
```bash
# Перейдите в папку проекта
cd "AutoScaleX Pro 2.2"

# Запустите скрипт установки
install.bat
# или для PowerShell:
.\install.ps1
```

**Вариант 2 (вручную):**
```bash
# Перейдите в папку проекта
cd "AutoScaleX Pro 2.2"

# Установите зависимости
pip install -r requirements.txt
```

**Вариант 3 (если проблемы с путями):**
```bash
pip install "requests>=2.31.0,<3.0" "python-telegram-bot>=20.0,<22.0" "python-dotenv>=1.0,<2.0" "cryptography>=41.0.0"
```

3. Создайте файл `.env` на основе `env.example`:
```bash
# Windows
copy env.example .env

# Linux/Mac
cp env.example .env
```

4. Заполните `.env` файл:
- `TG_TOKEN` — токен Telegram бота (обязательно)
- `ENCRYPTION_SECRET` — секрет для шифрования API ключей (обязательно, без него бот не запустится)
- `TG_ADMIN_ID` — ваш Telegram ID для админки (число; пустое значение = 0)
- `BINGX_API_KEY` и `BINGX_SECRET` — опционально, можно задать через бота

## Запуск

### Быстрый запуск (Windows):

1. Создайте файл `.env` из `env.example`:
   ```bash
   copy env.example .env
   ```

2. Заполните `.env` (TG_TOKEN и ENCRYPTION_SECRET обязательны)

3. Запустите бота:
   ```bash

   ```

   ```bash
   python main.py
   ```

```bash
cd "AutoScaleX Pro 2.2"
python main.py
```

Бот автоматически перезапустится при падении (до 10 попыток).

## Использование

### Для пользователей

1. Запустите бота командой `/start`
2. Введите ваш UID профиля (проверяется по списку рефералов)
3. Если реферал не найден, зарегистрируйтесь по ссылке из сообщения
4. Настройте API ключи командой `/set_api`
5. Используйте меню для управления ботом:
   - **Старт** — запуск торговли
   - **Пауза** — приостановка
   - **Стоп** — остановка и отмена ордеров
   - **Баланс** — просмотр баланса и прибыли
   - **Ордера** — количество открытых ордеров
   - **Настройки** — настройка параметров

### Настройки

- **Выбрать монету** — выбор торговой пары (BTC-USDT, ETH-USDT и т.д.)
- **Шаг сетки**: 0.75% (до 125 BUY, всего 130 ордеров) или 1.5% (до 60 BUY, всего 65 ордеров)
- **Размер ордера**: 10, 20, 35, 50, 75, 100 или 250 USDT
- **Построить сетку** — перестроить сетку вручную
- **Ребалансировать SELL** — перестроить SELL сетку от VWAP (3 ордера)

### Выбор торговой пары

Бот поддерживает **любую спотовую пару** на бирже BingX:
- Популярные пары: BTC-USDT, ETH-USDT, BNB-USDT, SOL-USDT и др.
- Можно выбрать из списка или ввести вручную
- При смене пары все ордера по старой паре автоматически отменяются
- Каждый пользователь может выбрать свою монету

### Для администратора

Используйте команду `/admin` для управления рефералами:
- Просмотр списка рефералов
- Добавление нового реферала
- Удаление реферала

📖 **Полный список команд:** см. файл `ADMIN_COMMANDS.md`

## Логика торговли

### Сетка и лимиты

- **Шаг 0.75%**: максимум 125 BUY ордеров, всего до 130 ордеров (125 BUY + 5 SELL)
- **Шаг 1.5%**: максимум 60 BUY ордеров, всего до 65 ордеров (60 BUY + 5 SELL)
- При старте: рыночная покупка на (5 × размер_ордера) + 2 USDT, затем 5 SELL выше цены и N BUY ниже (N по балансу и profit_bank)

### Исполнение BUY ордера

1. Позиция добавляется в FIFO очередь
2. **Сразу создаётся SELL ордер** на уровне +шаг от цены покупки (мультипликативно: `price × (1 + grid_step_pct)`)
3. Проверка критического уровня

### Исполнение SELL ордера

1. Прибыль считается по FIFO (от самой старой позиции)
2. Прибыль добавляется в `profit_bank`
3. **create_buy_after_sell**: после исполнения SELL создаётся один BUY снова на уровне покупки (ниже цены продажи на шаг: `sell_price × (1 - grid_step_pct)`); если эта цена занята — запасной шаг 1% (при 1.5%) или 0.5% (при 0.75%)
4. **Pyramiding**: пока `profit_bank >= размер_ордера`, создаются дополнительные BUY ниже (до лимита сетки)
5. **Подготовка к ребалансу**: если после исполнения остался только 1 открытый SELL — отменяются 5 самых низких BUY ордеров, чтобы освободить USDT для будущей новой SELL-сетки
6. Проверка ребалансировки (если все SELL закрыты)

### Защита большой сетки

- **Условие**: открытых BUY ≤ 3 и общее число открытых ордеров больше порога (62 при шаге 1.5%, 127 при шаге 0.75%)
- **Действие**: добавляется до 5 BUY ордеров внизу сетки (ниже самой низкой текущей BUY-цены), чтобы растянуть сетку вниз и снизить риск быстрого перехода в ребаланс при падении
- Проверка выполняется в основном цикле каждые 15 секунд

### Ребалансировка (0 SELL)

Когда все SELL ордера исполнены:
1. Рыночная покупка на (5 × размер_ордера) + 2 USDT
2. Создание 5 новых SELL ордеров выше текущей цены
3. BUY сетка не перестраивается

### Критический уровень

Когда все BUY ордера исполнены:
1. Бот переходит в состояние CRITICAL
2. Создаётся критическая SELL сетка (3 ордера от VWAP: +2%, +4%, +6%)
3. Уведомление о необходимости пополнения

### Ребалансировка SELL (ручная)

Команда «Ребалансировать SELL» создаёт 3 SELL ордера от VWAP:
- Уровни: VWAP × (1 + 2%), VWAP × (1 + 4%), VWAP × (1 + 6%)
- Весь объём распределяется равномерно

## Сохранение состояния

Все ордера и настройки сохраняются в файлы:
- `user_states/user_{user_id}.json` — состояние пользователя
- `user_data/<uid>.json` — сделки и profit_bank по UID
- `trades_{user_id}.csv` — история сделок
- `statistics_{user_id}.json` — статистика

При перезапуске бот автоматически восстанавливает состояние. При возобновлении с существующими ордерами на бирже FIFO-позиции восстанавливаются из истории сделок (`restore_from_trades`), чтобы расчёт прибыли по следующим SELL был корректным.

**Исправление ошибочного profit в истории:** скрипт `scripts/fix_profit_bank_ksm.py` — однократная правка файла сделок (обнуление ошибочного profit по одной SELL и пересчёт profit_bank). Запуск: `python scripts/fix_profit_bank_ksm.py path/to/user_data/<uid>.json`

## Безопасность

- API ключи хранятся в зашифрованном виде в файлах состояния
- Доступ только для проверенных рефералов
- Circuit breaker — защита от перегрузки API
- Rate limiting для запросов к бирже
