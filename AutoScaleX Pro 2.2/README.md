# AutoScaleX Pro 2.2

Спотовый торговый бот для BingX с Grid Trading, Pyramiding и DCA стратегиями.

## Особенности

- ✅ **Асинхронная архитектура** - быстрая и эффективная работа
- ✅ **Реферальная система** - доступ только для рефералов админского аккаунта
- ✅ **Полное Telegram управление** - все команды через удобный интерфейс
- ✅ **Автоматическое сохранение состояния** - восстановление после падения
- ✅ **Автоматический перезапуск** - устойчивость к сбоям
- ✅ **FIFO расчет прибыли** - точный учет позиций
- ✅ **Grid Trading** - автоматическая сетка ордеров
- ✅ **Pyramiding** - расширение сетки за счет прибыли
- ✅ **DCA** - усреднение позиций

## Установка

1. Клонируйте репозиторий или скопируйте файлы
2. Установите зависимости:

**Вариант 1 (рекомендуется для Windows):**
```bash
# Перейдите в папку проекта
cd "AutoScaleX Pro 2.2"

# Запустите скрипт установки
install.bat
# или для PowerShell:
.\install.ps1
```

**Вариант 2 (вручную):**
```bash
# Перейдите в папку проекта
cd "AutoScaleX Pro 2.2"

# Установите зависимости
pip install -r requirements.txt
```

**Вариант 3 (если проблемы с путями):**
```bash
pip install "requests>=2.31.0,<3.0" "python-telegram-bot>=20.0,<22.0" "python-dotenv>=1.0,<2.0" "cryptography>=41.0.0"
```

3. Создайте файл `.env` на основе `env.example`:
```bash
# Windows
copy env.example .env

# Linux/Mac
cp env.example .env
```

4. Заполните `.env` файл:
- `TG_TOKEN` - токен Telegram бота
- `TG_ADMIN_ID` - ваш Telegram ID (для админки)
- `BINGX_API_KEY` и `BINGX_SECRET` (опционально, можно установить через бота)

## Запуск

### Быстрый запуск (Windows):

1. Создайте файл `.env` из `env.example`:
   ```bash
   copy env.example .env
   ```

2. Заполните `.env` файл (TG_TOKEN и TG_ADMIN_ID обязательны)

3. Запустите бота:
   ```bash
   start.bat
   ```
   или
   ```bash
   python main.py
   ```

### Запуск из командной строки:

```bash
cd "AutoScaleX Pro 2.2"
python main.py
```

Бот автоматически перезапустится при падении (до 10 попыток).

## Использование

### Для пользователей

1. Запустите бота командой `/start`
2. Введите ваш UID профиля (проверяется по списку рефералов)
3. Если реферал не найден, зарегистрируйтесь по ссылке из сообщения
4. Настройте API ключи командой `/set_api`
5. Используйте меню для управления ботом:
   - **Старт** - запуск торговли
   - **Пауза** - приостановка
   - **Стоп** - остановка и отмена ордеров
   - **Баланс** - просмотр баланса и прибыли
   - **Ордера** - количество открытых ордеров
   - **Настройки** - настройка параметров

### Настройки

- **Выбрать монету** - выбор торговой пары (BTC-USDT, ETH-USDT и т.д.)
- **Шаг сетки**: 0.75% или 1.5%
- **Размер ордера**: 10, 20 или 50 USDT
- **Построить сетку** - перестроить сетку вручную
- **Ребалансировать SELL** - перестроить SELL сетку от VWAP (3 ордера)

### Выбор торговой пары

Бот поддерживает **любую спотовую пару** на бирже BingX:
- Популярные пары: BTC-USDT, ETH-USDT, BNB-USDT, SOL-USDT и др.
- Можно выбрать из списка или ввести вручную
- При смене пары все ордера по старой паре автоматически отменяются
- Каждый пользователь может выбрать свою монету

### Для администратора

Используйте команду `/admin` для управления рефералами:
- Просмотр списка рефералов
- Добавление нового реферала
- Удаление реферала

📖 **Полный список команд:** см. файл `ADMIN_COMMANDS.md`

## Логика торговли

### Инициализация

При запуске бот создает сетку ордеров:
- **BUY ордера**: 15-130 штук (зависит от profit_bank)
- **SELL ордера**: 5 штук
- Шаг между ордерами: настраивается (0.75% или 1.5%)

### Исполнение BUY ордера

1. Позиция добавляется в FIFO очередь
2. **Сразу создается SELL ордер** на уровне +шаг от цены покупки
3. Проверка критического уровня

### Исполнение SELL ордера

1. Прибыль рассчитывается по FIFO (от самой старой позиции)
2. Прибыль добавляется в `profit_bank`
3. **Pyramiding**: если `profit_bank >= размер_ордера`, создается новый BUY ниже
4. Проверка ребалансировки (если все SELL закрыты)

### Ребалансировка

Когда все SELL ордера исполнены:
1. Рыночная покупка на 5 ордеров
2. Создание новых 5 SELL ордеров
3. Перестройка BUY сетки

### Критический уровень

Когда все BUY ордера исполнены:
1. Бот переходит в состояние CRITICAL
2. Создается критическая SELL сетка (3 ордера от VWAP)
3. Уведомление о необходимости пополнения

### Ребалансировка SELL (ручная)

Команда "Ребалансировать SELL" создает 3 SELL ордера от VWAP:
- Уровни: VWAP × (1 + 2%), VWAP × (1 + 4%), VWAP × (1 + 6%)
- Весь объем распределяется равномерно

## Сохранение состояния

Все ордера и настройки сохраняются в файлы:
- `user_states/user_{user_id}.json` - состояние пользователя
- `trades_{user_id}.csv` - история сделок
- `statistics_{user_id}.json` - статистика

При перезапуске бот автоматически восстанавливает состояние.

## Безопасность

- API ключи хранятся в зашифрованном виде в файлах состояния
- Доступ только для проверенных рефералов
- Circuit breaker защита от перегрузки API
- Rate limiting для запросов к бирже

## Логирование
